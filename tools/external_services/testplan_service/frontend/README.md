# 3GPP Test Management Platform - Frontend

A professional, modern frontend application for 3GPP test management, built with vanilla JavaScript and featuring enterprise-grade UI/UX design.

## 🚀 功能特性

### 核心功能
- **用户认证系统**: 支持用户登录/登出，演示登录功能
- **Dashboard概览**: 实时显示测试执行状态和统计信息
- **测试计划管理**: 创建、编辑、删除和执行测试计划
- **文件上传**: 支持XLSX文件上传并自动生成测试计划
- **数据导出**: 支持导出为XLSX、CSV、PDF格式
- **实时刷新**: 自动刷新Dashboard数据
- **响应式设计**: 适配桌面、平板和手机设备

### 技术特性
- **现代化UI**: 基于CSS Grid和Flexbox的响应式布局
- **模块化架构**: 清晰的代码组织和模块分离
- **API集成**: 完整的RESTful API客户端
- **错误处理**: 统一的错误处理和用户反馈
- **性能优化**: 防抖、节流、虚拟化等性能优化技术
- **无障碍设计**: 支持键盘导航和屏幕阅读器

## 🎯 页面说明

### 登录页面 (`loginPage`)
- **功能**: 用户身份验证
- **特性**: 
  - 支持用户名/密码登录
  - 演示登录功能（用户名：demo，密码：demo123）
  - 表单验证和错误提示
  - 响应式设计

### Dashboard页面 (`dashboardPage`)
- **功能**: 测试执行状态概览
- **特性**:
  - 实时统计卡片（进行中、已完成、失败、总计）
  - 最近测试列表
  - 测试状态过滤
  - 自动刷新（30秒间隔）
  - 测试进度可视化

### 测试详情页面 (`testDetailPage`)
- **功能**: 测试计划管理
- **特性**:
  - 文件上传区域（支持拖拽）
  - 测试计划列表展示
  - 测试用例详情表格
  - 导出功能（XLSX、CSV、PDF）
  - 搜索和过滤功能

## 🛠️ 技术实现

### 架构设计
- **MVC模式**: 清晰的模型-视图-控制器分离
- **模块化**: 每个功能模块独立管理
- **事件驱动**: 基于事件的组件通信
- **状态管理**: 集中式状态管理

### 核心类说明

#### `Application`
- 主应用程序类，负责整体初始化和模块管理
- 处理全局事件和路由
- 管理页面切换和状态同步

#### `AuthManager`
- 认证管理，处理登录/登出逻辑
- Token管理和自动刷新
- 权限验证和用户状态维护

#### `DashboardManager`
- Dashboard数据管理和显示
- 统计数据实时更新
- 自动刷新和状态监控

#### `TestPlanManager`
- 测试计划CRUD操作
- 文件上传和解析
- 导出功能实现

#### `UIComponents`
- 通用UI组件管理
- Toast通知系统
- 模态框和对话框
- 加载状态和进度条

### API集成
- **MockAPIClient**: 开发环境的模拟API客户端
- **APIClient**: 生产环境的真实API客户端
- 支持认证、请求拦截、错误处理
- 自动token刷新机制

## 🔧 开发指南

### 本地开发
1. 克隆项目到本地
2. 使用HTTP服务器运行（如Python的`http.server`或Node.js的`http-server`）
3. 访问 `http://localhost:8000`

```bash
# 使用Python HTTP服务器
cd frontend
python -m http.server 8000

# 或使用Node.js http-server
npx http-server -p 8000
```

### 演示模式
应用自动检测运行环境：
- **开发环境**: 使用MockAPIClient，提供模拟数据
- **生产环境**: 使用真实的API客户端

### 快捷键
- **Ctrl/Cmd + K**: 全局搜索
- **Ctrl/Cmd + R**: 刷新页面数据
- **Esc**: 关闭模态框
- **F1**: 显示帮助

## 🐛 故障排除

### 常见问题
1. **页面空白**: 检查控制台错误，通常是JavaScript加载失败
2. **API调用失败**: 检查网络连接和API服务状态
3. **文件上传失败**: 检查文件格式和大小限制
4. **样式异常**: 检查CSS文件是否正确加载

### 调试技巧
- 打开浏览器开发者工具
- 查看控制台日志
- 使用网络面板检查API请求
- 检查本地存储中的认证信息
